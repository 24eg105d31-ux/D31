# Campus Resource Management System (CRMS)

A web-based Campus Resource Management System developed as a mini project to manage classrooms, labs, halls, and bookings efficiently.

---

## 📌 Project Overview

CRMS allows Students, Faculty, and Admins to manage campus resources through a role-based interface.

The system ensures:
- Controlled booking workflow
- Role-based access
- Conflict prevention
- Maintenance tracking
- Resource utilization monitoring

---

## 👥 User Roles & Functionalities

### 🎓 Student
- View available resources
- Request bookings
- View own bookings
- Cancel own bookings
- Raise maintenance requests

### 👨‍🏫 Faculty
- View available resources
- Create course-related bookings
- Manage own bookings
- View department resource utilization
- Raise maintenance requests

### 🛠 Admin
- Manage resources (Add / Edit / Delete)
- Approve or reject booking requests
- View all bookings
- Monitor high-demand resources
- Manage maintenance requests
- View system analytics

---

## 🔐 Role-Based Access Control

The system uses session-based role detection to:
- Show/hide menu items
- Restrict unauthorized actions
- Filter data visibility based on role
